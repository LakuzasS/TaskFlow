backend.auth.login_with_google(id_token_str)[source]

Connecte un utilisateur avec un token ID Google en vérifiant ses informations et en l’ajoutant à la base de données si nécessaire.

Cette fonction vérifie l’authenticité du token ID fourni, extrait les informations utilisateur (nom et e-mail) et tente de récupérer les informations associées à cet e-mail depuis la base de données. Si l’utilisateur n’existe pas, il est créé et un e-mail de confirmation est envoyé.

Parameters:

id_token_str (str) – Le token ID Google à vérifier.

Returns:

True si l’utilisateur est authentifié ou créé avec succès, False sinon.

Return type:

bool

backend.auth.send_confirmation_email(to_email, username)[source]

Envoie un e-mail de confirmation à l’utilisateur après la création de son compte.

Cette fonction utilise l’API Gmail pour envoyer un e-mail contenant un message de confirmation de création de compte à l’utilisateur spécifié. Elle s’assure également que le token d’authentification est valide avant d’envoyer l’e-mail.

Parameters:

  • to_email (str) – L’adresse e-mail de l’utilisateur à qui envoyer le message de confirmation.

  • username (str) – Le nom de l’utilisateur, utilisé dans le corps de l’e-mail.

Returns:

None

Ce décorateur est utilsé pour chaque requête SQL vers la base de données. Il crée automatiquement le curseur, puis le close une fois la fonction terminée. Utilisé en tant que “context manager”. :return: Le résultat de la fonction enfante ou une Exception en cas d’erreur

Envoie un email à un destinataire avec un sujet et un corps donnés.

Cette fonction utilise un compte de messagerie parmi ceux définis dans la liste accounts. Elle sélectionne le compte suivant dans la liste à chaque envoi, en utilisant un mécanisme de rotation.

backend.TaskFlow.check_tasks()[source]

Vérifie les tâches en fonction de leur date d’échéance et envoie des rappels par email.

Cette fonction vérifie les tâches dans la base de données pour déterminer si leur date limite est passée ou approche. Si la tâche est en retard, un email est envoyé à l’utilisateur assigné. Si la date limite approche, un rappel est envoyé.

backend.TaskFlow.main()[source]

Affiche un menu principal et permet d’exécuter diverses actions, telles que la gestion des utilisateurs, projets et tâches.

Cette fonction propose à l’utilisateur un menu interactif pour créer des utilisateurs et des projets, inviter des utilisateurs, ajouter des tâches, vérifier les tâches en retard ou envoyant des rappels.

backend.TaskFlow.update_task_info(task_info_id, status=None, priority=None, title=None, description=None, dead_line=None, covered=None, assign=None)[source]

Met à jour une tâche dans la table tasks_infos.

Cette fonction renvoie un boolean. En cas d’échec, elle renverra False et en cas de succes, elle renverra True.

Parameters:

  • task_info_id (int) – ID de la tâche à modifier.

  • status (int | None) – (optionnel) Nouveau statut de la tâche.

  • priority (int | None) – (optionnel) Nouvelle priorité de la tâche.

  • title (str | None) – (optionnel) Nouveau titre de la tâche.

  • description (str | None) – (optionnel) Nouvelle description de la tâche.

  • dead_line (datetime | None) – (optionnel) Nouvelle échéance (format datetime).

  • covered (bool | None) – (optionnel) Nouvel état “covered” (booléen).

  • assign (int | None) – (optionnel) Nouvel assigné (ID de tasks_groups).

Exemple: >>> success = update_task_info(task_info_id=4, title=’nouveau title’, priority=2, assign=2) >>> if success: >>> print(“supprimé avec succes”) >>> success = update_task_info(task_info_id=4, status=2) >>> if success: >>> print(“supprimé avec succes”)
